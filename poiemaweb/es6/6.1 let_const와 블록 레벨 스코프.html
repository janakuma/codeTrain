1. let

블록 레벨 스코프를 따르지 않는 var 키워드의 특성 상, 코드 블록 내의 변수는 전역변수이다. 이미 코드 블록 내에 전역변수가 선언되었다면 새로운 값을 재할당하여 덮어쓴다. let키워드로 선언된 변수는 블록 레벨 스코프를 따른다. 블록 레벨 스코프를 갖는 지역 변수는 전역에서 변수를 참조 할 수 없다. 1.2 변수 선언 금지 var 키워드는 동일한 이름을 갖는 변수를 중복해서 선언할 수 있었다. 하지만 let 키워드로는 동일한 이름을 갖는 변수를 중복해서 선언할 수 없다. 중복 선언시 에러가 발생한다. 
var foo = 123;
var foo = 456; //중복 선언 허용

let bar = 123;
let bar = 456; // Uncaught SyntaxError: Identifier 'bar' has already been declared
const는 선언과 동시에 할당이 이루어져야 한다. 또한 const는 let과 마찬가지로 블록 레벨 스코프를 갖는다. const는 재할당이 금지된다. 하지만 객체의 내용은 변경(프로퍼티의 추가, 삭제, 프로포티의 값의 변경)이 가능하다. 객체 타입 변수 선언에서는 const를 사용하는 것이 좋다. 명시적으로 객체타입변수의 주소값을 변경 하여야 한다면 let을 사용한다.

3. var vs let vs const

es6를 사용한다면 var키워드는 사용하지 않는다. 재할당이 필요한 변수에는 let을 사용한다. 변경이 발생하지 않는 (재할당이 필요 없는) 원시 타입과 객체 타입에는 const를 사용한다.
